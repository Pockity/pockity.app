It's finally here! AppleScript support for the Pockity Mac app is now in public release, and brings powerful automations around your expenses to your Mac. There's also a new MCP server implementation built on top if you'd like to use Pockity from your favourite LLM CLI agent. 

All the documentation for these is [here](https://faq.pockity.app/scripting/), and it's very easy to get started using the samples from the documentation.

There have been several people asking me if Pockity will support automated expense imports in a future release. Well, this release unlocks even more things that can be automated in Pockity, not just expenses. As this is optional, users who may choose to do it are definitely encouraged to use it, while others can continue to use Pockity to build their habits around mindful spending and budgeting. 

Go ahead and download the latest release from the [Mac App Store](https://apps.apple.com/app/apple-store/id1475098830?pt=119194029&ct=website&mt=8) and start making your money work for you, automatically. 

<hr />

#### New 
- Major enhancements to Natural Language Processing:
    - Added support for extracting recurring intents (e.g., "every 3 days", "monthly", "every other week").
    - Synchronized NLP recurring data with the entry editor to automatically populate the repeat schedule (RRULE).
    - Improved parsing reliability for simple merchant-amount entries like "1099 Internet".
    - Fixed character offset issues in NLP extraction for strings with leading/trailing whitespace.
    - Added a robust tag-prediction fallback to handle edge-case model classifications.

#### Bug Fixes
- Fixed a critical App Hang in the sidebar caused by performing a synchronous data fetch on the main thread during UI evaluation.
